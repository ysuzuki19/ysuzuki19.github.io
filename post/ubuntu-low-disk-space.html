ubuntuでlow disk spaceを解消する

ubuntudisk

June 20, 2021

はじめに

Ubuntu 起動時にlow disk spaceが出てくるやつ・・・

デスクトップに表示されるポップアップには，対処法の一つとして，Trash のクリーンなどが挙げられる．

実際，Trash をクリーンしたが，ポップアップは消えなかった．

そこで，Disk の使用率や内訳を調べてみたところ，困ったことになっていた．

ディスク使用率の確認

Process や Resources などを監視するのによく使うSystem Monitorについて，今回初めて知ったが，File Systemも見ることができるとのこと．

よく見たら，普通にFile Systemタブがあった．

確認したところ，99%であった(やばい)．

ディスク使用内訳を確認

自分のコンピュータの使い方的に，これほどディスクを使っている感覚がなかった．「なんでこんなに使ってるんだろう」と思って，ディスクの使用内訳を確認することにした．

Disk Usage Analyzerというアプリケーションを起動し，解析したい Disk を選択した後，数分待機したら，Disk 内訳を確認できる．

私の場合は，赤の部分はhomeで（こうなるのは正常と思われる）あった．そこで，容量が重く，かつ，特に取り急ぎ必要ではないものを探したが，どうやら，UE のビルドが非常に大きいということがわかった．ちなみに，UE に当たる部分は，以下のポインタの指す位置である．

UE は Ubuntu 上でビルドしているので，ソースコード毎ローカルに配置している．そういうのも理由の一つなのかなと推測し，もう少し探ってみた．

先程マウスポインタが指していたエリアをクリックすると，更にその中の使用率を確認できる．

その結果，バイナリやビルドと同じ程度の Source が確認され，その中でも Third Party が非常に大きかった．

解決策を考える

最近，UE をほとんど触っていない事もあり，UE をアンインストールすることにした．

まずは，UnrealEngine のディレクトリを削除する前に，どこに配置されているかを再確認する．

ランチャーから起動できるように，~/.local/share/application/にショートカットを配置していたりとかを確認した．

これをやらずにディレクトリだけを削除すると，空のショートカットが残ってしまい，気分が悪い．

UE の起動にはこれしか使っていないっぽいので，あとは普通に削除した（bin とか lib とか var とかに見つけたら後から消す！）．

最終的に

ディスク使用率が99%から66%まで下がった．

また，ディスク内訳も以下のようになり，先程まで多くの領域を専有していた部分が開放された．

おわりに

ディスク内訳を見て，必要以上に容量を使っているリソースが無い場合は，別付けのディスクをつける必要があるのかなと思った．

もしくは，古いリソースを外部ストレージに保存するのも一つの手段だと思った．
